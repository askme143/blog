# 줄바꿈 인식하기

한국어 문장이 줄바꿈이나 PDF/OCR 추출 과정에서 두 조각으로 나뉘었을 때, 두 조각 사이에 공백을 넣어야 하는지 그대로 붙여야 하는지 판단하는 문제입니다.

## 1. 줄바꿈 인식 문제

텍스트 추출 결과에서는 원래 줄바꿈 위치가 문장 내부에 섞여 들어옵니다.
이때 줄의 끝과 다음 줄의 시작을 단순히 공백으로 이어 붙이면 단어 내부가 깨지고, 반대로 무조건 붙이면 원래 띄어쓰기가 있던 위치가 사라집니다.

예를 들어 다음 두 경우는 서로 다른 처리가 필요합니다.

```text
한국어를 + 잘한다       -> 한국어를 잘한다
줄바꿈을 인식하 + 는 로직 -> 줄바꿈을 인식하는 로직
```

첫 번째는 어절 사이이므로 공백이 필요하고, 두 번째는 한 어절이 줄바꿈 때문에 쪼개진 것이므로 공백 없이 붙여야 합니다.

## 2. 해결 방법

형태소 분석기 Kiwi를 사용해 두 후보 문장을 비교합니다.

1. `prev + next`
2. `prev + " " + next`

`Kiwi.analyze()`의 목적 함수 점수를 비교합니다. 점수가 더 높은 쪽을 더 자연스러운 문장으로 보고, 그에 따라 공백 삽입 여부를 결정합니다.
즉, 줄바꿈으로 나뉜 두 조각을 공백 없이 붙인 후보와 공백으로 이은 후보 중 형태소 분석기가 더 그럴듯하다고 평가한 쪽을 선택합니다.

## 3. 결과

자체 수집한 샘플 데이터 기준 214 케이스 중 212 건에 대해 정답을 맞춰서 99.07% 정확도를 보였습니다.

특이한 점은 공백을 삽입해야 하는 문장 테스트 케이스에서 이어붙인 문장과 올바르게 띄어 쓴 문장에 대해 각각 목적 함수 점수를 산출하면 값이 같게 나온다는 것입니다.
이는 어찌보면 자연스러운 결과인데, 형태소 분석기는 붙여쓴 문장에 대해서도 올바르게 형태소 분석을 할 수 있어야 하기 때문입니다.
붙여 써야 하는 문장인 경우, 경계의 단어를 띄어 쓰면 둘이 별개의 단어가 되는 경우가 많은데, 이러면 점수가 달라집니다.
